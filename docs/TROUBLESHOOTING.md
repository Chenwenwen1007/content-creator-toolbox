# 排查手册

## 1. 解析失败

### 现象

- 返回 `VIDEO_NOT_FOUND`

### 常见原因

- 平台分享页结构更新，当前提取规则失效
- 分享内容不是视频，而是图文、直播或合集页
- 小红书页面触发风控，返回降级页或空白页

### 解决方法

- 先在浏览器中打开分享链接，确认它确实是公开视频
- 换用最新复制出来的分享文案重新解析
- 在后端日志中查看 `resolved_url`
- 将失败页面的 HTML 保存下来，对照 `server/parsers/` 更新规则

## 2. 返回 `UNSUPPORTED_PLATFORM`

### 常见原因

- 输入内容中的第一条链接不是视频平台链接
- 分享文案里混入了其他跳转地址

### 解决方法

- 尽量只粘贴一条完整的视频分享文案
- 确认域名属于抖音、快手、小红书

## 3. 小程序连接失败

### 现象

- `request:fail`
- `downloadFile:fail`

### 常见原因

- 本地调试地址写错
- 服务端没启动
- 微信开发者工具未关闭合法域名校验
- 线上环境尚未配置 HTTPS 域名

### 解决方法

- 本地开发时确认 [config.js](/F:/Pyhton_Project/WeChatProject/qushuiyin/config.js) 为 `http://127.0.0.1:8000`
- 启动后先访问 `http://127.0.0.1:8000/health`
- 微信开发者工具中关闭合法域名校验，仅用于本地调试
- 上线时改成你自己的备案 HTTPS 域名

## 4. 视频能解析但无法保存

### 常见原因

- 未授予相册权限
- 视频源地址已过期
- 服务器下载代理超时

### 解决方法

- 在微信设置中打开相册权限
- 重新点击一次解析，拿最新下载地址
- 查看后端日志中是否存在 `REMOTE_HTTP_ERROR`
- 适当增大 `.env` 中的 `DOWNLOAD_TIMEOUT`

## 5. ECS 上外网访问不到

### 常见原因

- 阿里云安全组未放行 `80/443`
- Nginx 未启动
- 域名未解析到 ECS 公网 IP
- HTTPS 证书未正确配置

### 解决方法

- 在安全组中放行 `22`、`80`、`443`
- 执行 `sudo systemctl status nginx`
- 用 `ping your-domain.com` 或 `nslookup your-domain.com` 检查域名解析
- 用 `sudo nginx -t` 检查 Nginx 配置
